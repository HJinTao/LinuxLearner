# Vol.6 Solution (Permissions)

## 任务 1: 让脚本跑起来
**命令:**
```bash
chmod +x deploy.sh
```
**解析:**
`+x` 表示添加执行权限 (eXecute)。也可以使用 `chmod 755 deploy.sh`，但 `+x` 更语义化，且保留了原来的读写权限。

## 任务 2: 保护私钥
**命令:**
```bash
chmod 600 id_rsa
```
**解析:**
`600` 表示所有者 (User) 可读写 (rw- = 4+2 = 6)，其他人没有任何权限 (--- = 0)。这是 SSH 私钥的标准权限要求。

## 任务 3: 开放 Web 目录
**命令:**
```bash
chmod 755 public_html
```
**解析:**
`755` 表示所有者拥有完全权限 (rwx = 7)，而组和其他人拥有读取和执行权限 (r-x = 5)。对于目录，`x` 权限意味着允许进入目录，这对于 Web 服务器读取其中的文件是必须的。

## 避坑指南
*   不要随意给 `777` 权限！这会让任何用户都可以修改你的文件，极不安全。
*   如果只是想给脚本添加执行权限，优先使用 `chmod +x`。
