# Chapter 02: 空间治理 (Space Governance)

## 1. 实战背景 (The Pain Point)

磁盘空间报警是每个服务器管理员的噩梦。

*   **场景一 (Log Rotation)**: 你的应用每天产生 1GB 的日志。如果不定期清理，硬盘一周就满了。你需要编写一个脚本，自动删除 7 天前的旧日志，但**绝对不能**删错系统关键日志（如 `auth.log`）。
*   **场景二 (Disk Hog Analysis)**: 你的用户目录 `home/user` 占用了 100GB 空间。你需要快速找出到底是哪个子目录（是 `Videos` 还是 `.cache`？）占用了最多空间。
*   **场景三 (Whitespace Hell)**: 你从网盘下载了一堆文件，文件名里全是空格（例如 `My Resume Final v2.pdf`）。当你试图用脚本批量处理它们时，空格把文件名截断了，导致脚本报错甚至误删文件。

本章我们将学习如何优雅地查找、分析和批量处理文件，哪怕它们的文件名很“恶心”。

## 2. 核心工具箱 (The Toolkit)

### 2.1 `find` - 精确制导导弹
`find` 是 Linux 中最强大的文件查找工具。
语法: `find [查找路径] [查找条件] [执行动作]`

**详细参数说明**:

| 参数 | 含义 | 示例 |
| :--- | :--- | :--- |
| `-name` | **按文件名查找**。支持通配符 `*`。 | `find . -name "*.log"` (找当前目录下所有 .log) |
| `-type` | **按文件类型查找**。`f`=普通文件, `d`=目录。 | `find . -type d` (只找目录) |
| `-mtime` | **按修改时间查找** (Modify Time)。单位是**天**。 | `+7` (超过7天), `-7` (7天内), `7` (正好7天前) |
| `-size` | **按文件大小查找**。 | `find . -size +100M` (大于100MB的文件) |
| `-print0` | **输出以 null 分隔**。默认是用换行符分隔文件名，但如果文件名里有空格，换行符会造成歧义。`-print0` 用 null 字符 (ASCII 0) 分隔，配合 `xargs -0` 使用。 | `find . -name "*.pdf" -print0` |
| `!` | **逻辑非 (取反)**。 | `find . ! -name "auth.log"` (除了 auth.log 以外的文件) |

**组合技**:
`find var/log -name "*.log" -mtime +7 ! -name "auth.log"`
(查找 `var/log` 下，名字是 .log，且修改时间超过 7 天，且名字**不是** auth.log 的文件)

### 2.2 `du` - 磁盘用量分析 (Disk Usage)
用于查看文件和目录占用了多少磁盘空间。

**详细参数说明**:

| 参数 | 英文全称 | 含义 | 示例 |
| :--- | :--- | :--- | :--- |
| `-h` | **h**uman-readable | **人类可读格式**。自动将字节转换为 K, M, G 等单位。 | `du -h` |
| `-s` | **s**ummary | **只显示汇总**。不列出每个子文件的详情，只显示指定目录的总大小。 | `du -sh .` (看当前目录总大小) |
| `-d <N>` | **d**epth | **指定目录深度**。只显示到第 N 层子目录。 | `du -h -d 1 .` (只看第一层子目录大小) |

**组合技 (找出谁是吃硬盘怪兽)**:
`du -h -d 1 . | sort -hr`
*   `du -h -d 1 .`: 列出当前目录下一级子目录的大小。
*   `sort -hr`:
    *   `-h`: 识别 K, M, G 单位进行排序 (Human numeric sort)。
    *   `-r`: 倒序 (Reverse)，让最大的排在最上面。

### 2.3 `xargs` - 参数传递流水线
`find` 找到文件后，通常需要传给 `rm` 或 `mv` 等命令处理。`xargs` 就是这个传递者，它能把标准输入 (Stdin) 转换成命令行参数 (Arguments)。

**为什么需要它？**
很多命令（如 `rm`, `mv`）不支持直接通过管道接收文件名。你不能写 `find . -name "*.tmp" | rm`。
你需要写 `find . -name "*.tmp" | xargs rm`，它的效果等同于 `rm file1.tmp file2.tmp ...`。

**详细参数说明**:

| 参数 | 含义 | 示例 |
| :--- | :--- | :--- |
| `-0` | **处理 null 分隔符**。当输入是由 `find -print0` 产生时，必须加上这个参数。这是处理带空格文件名的**唯一正确姿势**。 | `find . -print0 | xargs -0 rm` |
| `-I {}` | **指定占位符**。如果你要把文件名插入到命令的中间（而不是末尾），就用这个。 | `find . -name "*.txt" | xargs -I {} mv {} backup/` (把找到的文件移到 backup 目录) |

## 3. 实验任务 (Hands-on Lab)

请先在 `Chapter_02` 目录下运行初始化脚本：
```bash
bash lab/init.sh
```
这会生成 `var/log` (日志), `home/user` (用户数据) 和 `downloads/` (乱七八糟的文件)。

### 任务 1: 智能日志清理 (Smart Cleanup)
**目标**: 清理 `var/log` 下的旧日志，释放空间。
*   **要求**: 删除所有修改时间超过 **7天** (`+7`) 的文件。
*   **约束**: **绝对不能**删除名为 `auth.log.old` 的文件（假设它是审计留存）。
*   *提示*:
    1.  构造 `find` 命令：指定目录 `var/log`。
    2.  添加时间条件：`-mtime`。
    3.  添加排除条件：`! -name`。
    4.  最后通过管道传给 `xargs rm` 执行删除。

### 任务 2: 空间占用分析 (Disk Usage Report)
**目标**: 找出 `home/user` 目录下，哪个子目录占用空间最大。
*   **要求**: 分析 `home/user` 的 **一级子目录** 大小，并将结果保存到 `space_report.txt`。
*   *提示*:
    1.  使用 `du` 命令，记得加上 `-h` (人类可读) 和 `-d 1` (只看一层)。
    2.  通过管道传给 `sort` 命令，使用 `-hr` (按单位倒序) 排序。
    3.  重定向 `>` 到文件。

### 任务 3: 拯救空格文件 (Whitespace Rescue)
**目标**: `downloads/` 目录下有一堆文件名带空格的 PDF 文件（如 `Tax Return 2023.pdf`）。
*   **要求**: 将所有 `.pdf` 文件 **移动** 到 `documents/` 目录下。
*   **约束**: 必须使用命令行完成，且不能报错。
*   *提示*:
    1.  这是 `find -print0` 和 `xargs -0` 的绝佳练习场。
    2.  `mv` 命令如果要把多个文件移到指定目录，通常需要 `-t` 参数（target directory），例如 `mv -t 目标目录 文件1 文件2...`。
    3.  组合起来：`find ... -print0 | xargs -0 mv -t documents/`。

## 5. 进阶思考 (Deep Dive)

1.  **`-exec` vs `xargs`**: `find ... -exec rm {} \;` 会为每个文件启动一个 `rm` 进程，效率低。`xargs` 会把文件名攒成一批，一次性传给 `rm`，效率高得多。
2.  **inode 耗尽**: 有时候磁盘没满，但无法创建新文件。这可能是 inode 耗尽（通常是因为有几百万个小文件）。使用 `df -i` 查看。
3.  **大文件查找**: `find . -type f -size +100M` 可以快速定位大于 100MB 的文件。
