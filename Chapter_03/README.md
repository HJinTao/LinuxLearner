# Chapter 03: 性能审判 (Performance)

## 1. 实战背景 (The Pain Point)

**场景一**：周五下午 5 点，你的服务器风扇突然开始狂转，像直升机起飞一样。网页响应变得极慢。你必须在 1 分钟内找出是哪个程序在“吸血”（占用 CPU），并干掉它。

**场景二**：你启动了一个 Python 脚本跑数据，结果它卡住了，按 `Ctrl+C` 没反应。关闭终端窗口？不，它可能还在后台偷偷跑着。你需要精准找到它的进程 ID (PID) 并强制结束它。

本章我们将学习如何监控系统健康状态，并对异常进程执行“死刑”。

## 2. 核心命令详解 (The Toolkit)

### 2.1 `top` - 任务管理器
Linux 下实时的性能监控面板。
*   **PID**: 进程 ID (Process ID)，进程的身份证号。
*   **USER**: 谁启动的进程。
*   **%CPU**: CPU 占用率。
*   **%MEM**: 内存占用率。
*   **COMMAND**: 进程的名字。

> **操作技巧**:
> *   进入 `top` 后，按 `P` (大写) 按 CPU 排序。
> *   按 `M` (大写) 按内存排序。
> *   按 `q` 退出。

### 2.2 `ps` - 进程快照 (Process Status)
如果说 `top` 是录像，`ps` 就是拍照。
*   `ps aux`: 查看系统所有进程（标准姿势）。
*   `ps aux | grep python`: 配合 grep 查找特定的进程。

### 2.3 `kill` - 终止信号
*   `kill PID`: 发送 SIGTERM 信号（温柔地告诉进程：该结束了，请收拾一下退出）。
*   `kill -9 PID`: 发送 SIGKILL 信号（强制查杀：立刻消失！不管有没有保存数据）。**慎用**。

**终端 B (解决故障)**:

### 任务 1: 逮捕 CPU 刺客
**现象**: 你的电脑（或虚拟机）风扇可能开始转了。
**目标**: 使用 `top` 找到 `%CPU` 最高的那个 `python3` 进程。
**操作**:
1.  输入 `top`。
2.  观察前几行，找到 `python3` 或 `high_cpu.py`。
3.  记下它的 **PID**。
4.  按 `q` 退出 top。
5.  执行 `kill <PID>`。

### 任务 2: 清理僵尸服务
**现象**: 有个 `frozen_service.py` 在后台睡觉，虽然不占 CPU，但占着内存和坑位。
**目标**: 使用 `ps` 找到它并结束它。
**操作**:
1.  输入 `ps aux | grep frozen`。
2.  找到对应的 PID（注意：grep 命令本身也会产生一个进程，别杀错了）。
3.  执行 `kill <PID>`。

### 任务 3: 搜查伪装者
**现象**: 有个脚本 `sneaky_process.sh` 伪装得很好。
**目标**: 找到并强制杀死它。
**操作**:
1.  使用 `ps aux` 配合 `grep` 找到它。
2.  这次尝试使用 `kill -9 <PID>` 体验强制查杀的快感。

## 5. 避坑与原理 (Deep Dive)

1.  **PID 1**: 永远不要尝试杀掉 PID 为 1 的进程（通常是 `systemd` 或 `init`）。那是系统的祖宗，杀掉它系统就崩了（虽然通常你也杀不掉）。
2.  **僵尸进程 (Zombie)**: 有时候你看到进程状态是 `Z`，并在 `defunct`。这意味着进程已经死了，但它的父进程还没给它“收尸”（读取退出状态）。`kill` 僵尸是没用的，你需要 kill 它的父进程。
3.  **kill 信号**: `kill` 默认发送信号 15 (SIGTERM)。程序可以捕获这个信号并执行清理操作（比如保存文件）。`-9` (SIGKILL) 是内核级别的强制移除，程序没有任何反应机会。
