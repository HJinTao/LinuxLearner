# Chapter 03: 性能审判 (Performance)

## 1. 实战背景 (The Pain Point)

服务器突然变慢，网页打不开，SSH 登录卡顿。这时候，你需要像医生一样，快速诊断出是谁在“吸血”。

*   **场景一 (CPU Hog)**: 有个 Python 脚本写了个死循环，占用了 100% 的 CPU。你需要立刻找到它的 PID 并杀掉它。
*   **场景二 (Zombie Process)**: 一个父进程挂了，留下一堆“僵尸进程”占着资源不释放。
*   **场景三 (Unkillable Service)**: 你尝试用 `kill` 命令停止一个服务，但它毫无反应（可能是在处理数据，或者配置了信号屏蔽）。你需要强制终结它。

本章我们将学习如何监控系统状态，并对失控的进程进行“审判”。

## 2. 核心工具箱 (The Toolkit)

### 2.1 `top` - 实时监控仪表盘
`top` 是 Linux 下最常用的性能分析工具，类似 Windows 的任务管理器。

**关键指标解读**:

*   **Load Average**: 平均负载（1分钟、5分钟、15分钟）。
    *   *判断标准*: 如果这个数字超过了 CPU 核心数 (例如 4 核 CPU，负载超过 4)，说明系统过载了。
*   **%CPU**: 进程占用的 CPU 百分比。
*   **%MEM**: 进程占用的内存百分比。
*   **PID**: 进程 ID (Process ID)。杀进程就靠它。
*   **COMMAND**: 进程名称。

**交互快捷键** (在 top 界面中按):
*   `P`: **P**ercentage CPU。按 CPU 使用率排序 (默认)。
*   `M`: **M**emory。按内存使用率排序。
*   `k`: **k**ill。杀掉进程 (按下后输入 PID 回车)。
*   `q`: **q**uit。退出。

### 2.2 `ps` - 进程快照 (Process Status)
`top` 是动态的，`ps` 是静态的快照。

**最常用咒语**: `ps aux`
(BSD 风格参数，不需要加横杠 `-`)

| 参数 | 含义 |
| :--- | :--- |
| `a` | **a**ll users。显示所有用户的进程，而不仅仅是你自己的。 |
| `u` | **u**ser-oriented。以用户为主的格式显示 (显示 USER, %CPU, %MEM, START 等详细信息)。 |
| `x` | 显示没有控制终端 (tty) 的进程。通常指后台服务 (Daemons)。 |

**组合技**:
`ps aux | grep python`
(查找所有 Python 进程)

**System V 风格**: `ps -ef`
*   `-e`: Select **e**very process (等同于 `a`).
*   `-f`: **f**ull-format listing (完整格式).
*   有些人更喜欢用 `ps -ef`，效果差不多，看个人习惯。

### 2.3 `kill` - 终结者
用于向进程发送信号 (Signal)。不要被名字吓到，`kill` 不仅仅是杀进程，它其实是**发送信号**的工具。

语法: `kill [信号] <PID>`

**常用信号**:

| 信号编号 | 信号名称 | 含义 | 详细解释 |
| :--- | :--- | :--- | :--- |
| `-15` | **SIGTERM** | Terminate (终止) | **默认信号**。这是“礼貌”的请求。告诉进程：“请收拾一下，尽快退出”。进程可以捕获这个信号并执行清理操作（如保存文件、关闭连接），或者**忽略**它。 |
| `-9` | **SIGKILL** | Kill (强制杀) | **强制执行**。内核会直接从内存中抹除该进程。进程无法捕获，也无法忽略。**慎用！** 因为这可能导致数据丢失或文件损坏。 |
| `-2` | **SIGINT** | Interrupt (中断) | 等同于你在终端按 `Ctrl+C`。 |

**示例**:
*   `kill 1234`: 发送 SIGTERM 给 PID 1234 (先礼后兵)。
*   `kill -9 1234`: 发送 SIGKILL 给 PID 1234 (强制处决)。
*   `pkill -f "script_name"`: 按名称杀死进程 (不需要查 PID)。

## 3. 实验任务 (Hands-on Lab)

请先在 `Chapter_03` 目录下运行初始化脚本：
```bash
bash lab/init.sh
```
这会启动两个后台进程：一个伪装的“矿工”脚本，和一个“顽固”的服务脚本。

### 任务 1: 猎杀 CPU 刺客 (Kill the Miner)
**目标**: 找到并终止名为 `crypto_miner.sh` 的进程。
*   **步骤**:
    1.  使用 `ps aux | grep miner` 或 `pgrep -a miner` 找到它的 PID。
    2.  使用 `kill <PID>` 终止它 (默认信号即可)。
    3.  再次检查它是否消失。

### 任务 2: 终结不死之身 (Force Kill)
**目标**: 找到并终止名为 `invincible_service.sh` 的进程。
*   **步骤**:
    1.  找到它的 PID。
    2.  尝试使用普通的 `kill <PID>`。你会发现它在终端输出嘲讽信息："Ha! I refuse to die!"。
        *   *原理*: 这个脚本代码里捕获了 SIGTERM 信号，并打印了那句话。
    3.  使用终极武器 `kill -9 <PID>` 强制终结它。

## 5. 进阶思考 (Deep Dive)

1.  **htop**: 比 `top` 更现代、更好看的工具。支持鼠标操作，条形图显示。如果系统里有，强烈推荐使用。
2.  **优雅退出**: 开发程序时，应该捕获 `SIGTERM` 信号，在退出前保存数据、关闭连接。直接 `kill -9` 可能导致数据丢失或文件损坏。
3.  **Kill all**: `killall nginx` 可以杀死所有名为 nginx 的进程。
